**Date**: 2026-03-23
**Scope**: Exhaustive line-level comparison of Adapted FastGen vs Original FastGen, with OmniAvatar-Train as reference

## 1. Executive Summary

The adaptation is architecturally well-designed — it uses clean inheritance and composition rather than patching core FastGen files. All core training infrastructure (loss functions, EMA, gradient clipping, checkpointing, noise schedule, trainer loop) is **byte-identical** between the original and adapted repos.

However, the review identified **4 critical issues** that would prevent correct training, **1 new critical issue** found by the adversarial reviewer, and several high/medium-risk items that affect training quality or correctness.

## 2. Critical Issues (MUST FIX)

### CRIT-1: Dataloader not wired into experiment configs

**Files**: `fastgen/configs/experiments/OmniAvatar/config_sf.py`, `config_kd.py`
**Verified by**: Agent 4 (initial), Adversarial agent (confirmed)

Neither config assigns an OmniAvatar dataloader. The inheritance chain is:
```
config_sf.py → config_omniavatar_sf.py → config_self_forcing.py → config_dmd2.py → BaseConfig
```
`BaseConfig.dataloader_train` defaults to `CIFAR10_Loader_Config`. The configs only override `batch_size=1`, leaving the CIFAR-10 loader active.

**Impact**: Training will crash immediately trying to load CIFAR-10 data.

**Fix**: Add to `config_sf.py` and `config_kd.py`:
```python
from fastgen.datasets.omniavatar_dataloader import create_omniavatar_dataloader
# ... or create a LazyCall config for the OmniAvatar dataloader
config.dataloader_train = L(create_omniavatar_dataloader)(
    data_dir="<path>",
    mask_path="<path>",
    # ... other params
)
```

### CRIT-2: Fake_score will be 14B instead of intended 1.3B

**Files**: `fastgen/configs/experiments/OmniAvatar/config_sf.py` (line 48-59), `fastgen/methods/distribution_matching/dmd2.py` (line 48)
**Verified by**: Agent 4 (initial), Adversarial agent (confirmed)

`OmniAvatar_V2V_1_3B_FakeScore` is defined but **never assigned** to any config field. `DMD2Model.build_model()` instantiates fake_score from `self.teacher_config` (which resolves to `config.model.teacher` — the 14B model).

**Impact**: Fake_score will be a 14B model (~30 GB extra VRAM), contradicting the design intent (1.3B fake_score). This would likely OOM on the target 2x A100 setup.

**Fix**: The DMD2/SelfForcing framework needs to support a separate `fake_score` config, or `OmniAvatarSelfForcingModel` must override `build_model()`:
```python
def build_model(self):
    super().build_model()
    # Replace 14B fake_score with 1.3B
    self.fake_score = instantiate(OmniAvatar_V2V_1_3B_FakeScore)
    self.fake_score.load_state_dict(self.teacher.state_dict(), strict=False)
```

### CRIT-3: KD training will crash — `data["path"]` key missing

**Files**: `fastgen/methods/omniavatar_kd.py` (line 94), `fastgen/datasets/omniavatar_dataloader.py`
**Verified by**: Agent 4 (initial), Adversarial agent (confirmed)

`OmniAvatarKDModel.single_train_step()` accesses `data["path"]` for precomputed ODE trajectories, but `OmniAvatarDataset` does not produce this key. KD requires a separate dataloader that loads ODE trajectory `.pt` files generated by `scripts/generate_omniavatar_ode_pairs.py`.

**Impact**: KD training crashes with `KeyError: 'path'` at the first iteration.

**Fix**: Create an ODE-trajectory-aware dataset class or extend `OmniAvatarDataset` to load `ode_path.pt` files and return them under the `"path"` key.

---

### CRIT-4: Discriminator inner_dim mismatch with 14B teacher features

**Files**: `fastgen/configs/experiments/OmniAvatar/config_sf.py` (line 100), `fastgen/configs/discriminator.py`
**Found by**: Adversarial agent (new finding)

The config uses `Discriminator_Wan_1_3B_Config` which has `inner_dim = 1536 // 4 = 384`. But features are extracted from the 14B teacher (dim=5120), producing features of dimension `5120 // 4 = 1280` after unpatchification. The discriminator's first convolution layer expects 384-channel input but receives 1280-channel features.

**Impact**: Runtime shape mismatch error at first discriminator forward pass.

**Fix**: Either:
1. Use `Discriminator_Wan_14B_Config` (inner_dim=1280), or
2. Add: `config.model.discriminator.inner_dim = 1280`

---

### CRIT-5: QwenImage configs reference non-existent loader class

**Files**: `fastgen/configs/experiments/QwenImage/config_dmd2.py`, `config_sft.py`
**Found by**: Agent 4

Both import `HiDreamG5_JourneyDB_Loader_Config` from `fastgen.configs.data`, but this symbol does not exist. The original files imported `ImageLoaderConfig`.

**Impact**: `ImportError` when loading QwenImage configs (not OmniAvatar-specific, but a regression in the adapted repo).

**Fix**: Revert to `ImageLoaderConfig` or add the missing loader config class.

## 5. Architecture Comparison

### Component Mapping

| FastGen Component | Original (WanT2V) | Adapted (OmniAvatar) |
|-------------------|-------------------|---------------------|
| Teacher | Wan 1.3B (bidirectional) | **OmniAvatarWan 14B** (bidirectional) |
| Student | CausalWan 1.3B | **CausalOmniAvatarWan 1.3B** |
| Fake Score | Wan 1.3B (same as teacher) | **Should be OmniAvatarWan 1.3B** (BUG: currently 14B) |
| Discriminator | Wan_1_3B_Config (inner_dim=384) | **Same config** (BUG: inner_dim should be 1280) |
| Training Method | SelfForcingModel | **OmniAvatarSelfForcingModel** (overrides `_prepare_training_data` only) |
| KD Method | CausalKDModel | **OmniAvatarKDModel** (overrides condition building + train step) |
| Loss Functions | VSD + DSM + GAN (common_loss.py) | **Identical** (byte-identical file) |
| Data Pipeline | WebDataSet (video) | **OmniAvatarDataset** (precomputed latents) |
| Noise Schedule | RFNoiseSchedule | **Identical** (byte-identical file) |
| EMA | EMACallback | **Identical** (byte-identical file) |
| Trainer | Trainer.run() | **Identical** (6-line debug log removed, no functional change) |

### Conditioning Pipeline

```
OmniAvatar-Train (original):
  video → VAE.encode → latents [16, 21, 64, 64]
  audio → Wav2Vec2 → [81, 10752] → AudioPack → [32, 11, 1, 1]
  text → T5 → [512, 4096]
  mask → LatentSync PNG → [64, 64]

FastGen Adaptation:
  All precomputed and loaded from .pt files by OmniAvatarDataset
  Assembled into condition dict by _prepare_training_data()
  Passed to OmniAvatarWan._build_y() for channel concatenation
  Audio injected additively at DiT layers 2 through num_layers//2
```

## 8. Adversarial Review Supplementary Report

The adversarial reviewer **confirmed** 6 of 8 prior findings, **challenged** 2 (downgrading severity), and found **1 new critical issue** (discriminator inner_dim mismatch).

### Prior Findings Verification

| # | Finding | Verdict |
|---|---------|---------|
| 1 | Dataloader not wired | **CONFIRMED** |
| 2 | Fake_score 14B instead of 1.3B | **CONFIRMED** |
| 3 | KD `data["path"]` missing | **CONFIRMED** |
| 4 | `x = noise` vs scaled noise | **CHALLENGED** — severity downgraded. Only affects standalone inference (`sample()` method), not training. Training uses `noise_scheduler.latents()` correctly in `_generate_noise_and_time()`. The 0.1% scaling difference at t=0.999 is negligible. |
| 5 | WanI2V `_replace_first_frame` no-op | **CONFIRMED** |
| 6 | Per-frame vs per-chunk mask | **CONFIRMED** — intentional design choice |
| 7 | `mask[0]` batch assumption | **PARTIALLY CONFIRMED** — correct for current dataset (single fixed mask), fragile if extended |
| 8 | VAE `mode="argmax"` dropped | **CHALLENGED** — not applicable to OmniAvatar (uses precomputed latents, no runtime VAE encoding). Only affects non-OmniAvatar inference scripts. |

### New Findings from Adversarial Review

- **CRITICAL**: Discriminator inner_dim=384 vs 14B teacher feature dim=1280 (see CRIT-4 above)
- **HIGH**: Noise schedule shift=5.0 default vs teacher's shift=3.0 (see H6 above)
- **LOW**: No import shadowing detected — clean module boundaries
- **LOW**: Gradient flow through self-forcing rollout verified correct and preserved

### Areas Confirmed Safe

- All loss computations (VSD, DSM, GAN) — identical to original
- EMA update logic — byte-identical callback
- Gradient clipping — byte-identical callback
- Checkpoint save/load — byte-identical
- Optimizer setup in DMD2 — alternating student/fake_score updates preserved
- Self-forcing rollout gradient flow — detach boundaries correctly preserved
- Import boundaries — no shadowing between OmniAvatar and FastGen modules
